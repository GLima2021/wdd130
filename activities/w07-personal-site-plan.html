Your personal Site Plan

Just as we did for the rafting site, before we build our personal (final) project we need to do some planning. We started this last week by preparing our purpose. This week we will begin a Site plan for the personal site we want to build that is very similar to the one we prepared for the rafting site.

Let's start by creating a new repository for our final website. Open up github.com and login if you need to. Click the 'New repository' option from the + drop down in the upper right.

Give it a name and remember this will be a part of the URL (web address) so keep it short, simple, and professional. The name should also have something to do with the topic of your personal site. Don't use the example in the image below.

You can add a description if you'd like and leave the Public radio button selected and check the "Add a README file" checkbox. Then click 'Create respository'

You may see a screen like this:

Click the repo name in the upper left to see what is contained in the repo.

Now we need to tell VSCode and our Local Git about our repo. Click on the 'Code' drop down menu button and copy the HTTPS URL.

Open up VSCode and from the 'View' menu option choose 'Command Palette'.

Type git clone to bring up the 'Git: Clone' option in blue below it. Click that blue bar with the text 'Git: Clone'

You will see a place in the text box to paste your HTTPS URL that you copied earlier. Just click inside the textbox where it says "Provide respository URL ..." and paste the URL and hit enter

Your file management window will open and you can choose a location of where you want the repository on your local Git. Choose a location and click 'Select Repository Location'. It will create a folder in that location with the name of your repo and the only file in there right now is the README.md file.

If you get a pop-up asking if you want to open the cloned repository you can click 'Open' or just open it as File menu option 'Add Folder to Workspace'. You may also get a "Do you Trust these files" just say 'Yes, I Trust..'. If you get a question about saving you workspace configuration as a file you can just click 'Cancel'. But don't worry if you don't get some of these pop-ups, but do open the new repo folder in VSCode.

Now VSCode, our local Git and GitHub can all work together just as we've been doing with our rafting site repo. As you add and change files in this folder the changes will be noticed and wait to be committed and pushed to the GitHub repo.

Add a file to your new repo called site-plan.html.

Go to: Site Plan Template Right-click on the page and view the source code. (On Chrome it's the pop-up menu option 'View Page Source')

Copy all the html code. You can use Cmd+A or Ctrl+A to select it all quickly, then copy it.

Paste the code into your site-plan.html page in VSCode and save the file.

As you look at the new site plan template you will notice it is very similar to the last one. It does have a new section however...Content. For the Dry Oar site I provided you with content, you will need to generate all of the content (text and media) for your site.

You will also be required to create three wireframes. One for each page in your site.

We will have some activities designed to help you complete your site plan...but much of it you will need to do on your own. The completed site plan for your personal site is due at the end of Week 9. This gives you three weeks to complete it. Don't wait until the beginning of Week 9 to start!!

Requirements

  1. The Web site must consist of a home page and at least 2 child pages
  2. One of the child pages should incorporate a form.
  3. The Web site must be hosted on the web.
  4. Each page of your Web site must satisfy the following requirements:
    1. Significant textual content consisting of appropriate content for your audience so that they can understand the meaning and message of this site
    2. At least one image that relates to the content of the page
    3. Working links to your two child pages
    4. Be laid out and styled according to your site plan.
    5. Be visually appealing and easily readable
